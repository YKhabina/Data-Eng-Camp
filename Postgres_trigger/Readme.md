# Проект: Аудит изменений пользователей с автоматическим экспортом в CSV

## Техническое задание (ТЗ)

### Цель
Реализовать систему отслеживания изменений данных пользователей (поля `name`, `email`, `role`) в условиях отсутствия готового механизма аудита. Система должна автоматически логировать все изменения, хранить историю в отдельной таблице и ежедневно экспортировать свежие записи в CSV-файл с помощью планировщика `pg_cron`.

### Функциональные требования

1. **Логирование изменений**  
   - Отслеживать все операции вставки (`INSERT`), обновления (`UPDATE`) и удаления (`DELETE`) над таблицей `users`.
   - Фиксировать следующие данные:  
     - идентификатор пользователя (`user_id`),  
     - временная метка изменения (`changed_at`),  
     - старые и новые значения полей `name`, `email`, `role`.

2. **Хранение аудита**  
   - Хранить аудит изменений в отдельной таблице,
   - Создать отдельную таблицу `users_audit` для хранения истории изменений.  

3. **Механизм автоматического логирования**  
   - Реализовать триггер на таблице `users`, который при любом изменении (INSERT/UPDATE/DELETE) вызывает специальную функцию логирования и записывает запись в `users_audit`.

4. **Ежедневный экспорт в CSV**  
   - Создать функцию, которая выгружает **только свежие изменения** (за прошедший день) из таблицы `users_audit` в CSV-файл.  
   - Формат имени файла: `/tmp/users_audit_export_YYYY-MM-DD.csv`, где `YYYY-MM-DD` – дата, за которую произведён экспорт (вчерашний день).  
   - Файл должен содержать заголовки колонок.

5. **Автоматизация экспорта**  
   - Установить расширение `pg_cron` в PostgreSQL.  
   - Настроить планировщик на ежедневный запуск функции экспорта в **3:00 утра**.

6. **Среда исполнения**  
   - Все компоненты должны работать внутри Docker-контейнера на основе PostgreSQL.  
   - После выполнения всех команд (инициализация, запуск) в контейнере должен автоматически создаваться CSV-файл с изменениями за прошедший день.

### Требования к поставке

- Репозиторий Git должен содержать:  
  - `Dockerfile` для сборки образа PostgreSQL с необходимыми расширениями и скриптами инициализации.  
  - `docker-compose.yml` для удобного запуска контейнера.  
  - `init.sql` – скрипт инициализации базы данных, создающий таблицы, функции, триггер и настройки `pg_cron`.  
  - `README.md` (текущий документ) с чётким описанием ТЗ и инструкциями по сборке и запуску (без раскрытия кода решения).

### Критерии приёмки

- При внесении изменений в таблицу `users` соответствующая запись появляется в `users_audit`.
- Ежедневно в 3:00 утра (или при ручном вызове экспортной функции) в `/tmp/` контейнера появляется CSV-файл со всеми записями аудита за предыдущий день.
- CSV-файл имеет корректную структуру (заголовки, разделители, экранирование).
- Планировщик `pg_cron` настроен и работает без ошибок.

### Инструкция по сборке и запуску (общая)

1. Клонировать репозиторий.  
2. Выполнить `docker-compose up -d --build`.  
3. Подключиться к базе данных внутри контейнера и внести тестовые изменения в `users`.  
4. Проверить наличие CSV-файла после выполнения экспортной функции вручную или дождаться автоматического запуска.
